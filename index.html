DATA PROJECTS

Project Three | Web Scraping

What is Data Science without the data? Getting data from well curated sources, like pre-built flat files or an API, still requires processing and data wrangling. However, scraping sites for content can add another level of complexity. What happens if data is missing? What happens if a table moves? What happens if columns change order? How should data types be processed and stored? What happens if the site is down? The Python community has created several packages to help process data scraped from an HTML page. Here, I try out Beautiful Soup.

Keywords: Python, Web scraping

Project Two | Stock ETL

Stock data is fantastic data to use in practicing data handling, modeling and prediction. There are two general categories of stock data, historical and real-time. Models may be built using historical data and evaluated using real-time data. The primary purpose of this project is to practice building ETL pipelines that retain a clear data model and handle event errors appropriately. This is an ongoing project and serves as the foundation and data source for downstream modedling and event predictions.

Keywords: Python, SQL, AWS, API

Project One | Insight Demo

Fellows at Insight Data Science (see above for program details) are expected to rapidly build a unique and novel data project for presentation to companies. The goal is to demonstrate the ability to develop, from scratch, product insight from data. The specific project I worked on was fun and challenging and involved mining masked text messaging data for a type of sentiment. Although the code is very clearly not satisfactory for production, it sufficed in generating a proof-of-concept and clearly demonstrated strong structure in the data.

Keywords: R
